# AI 使用日志目录

本目录用于存储 MCM/ICM 竞赛期间的 AI 工具交互记录，以满足赛事合规要求。

## 日志格式规范

每次重要 AI 交互应记录为独立 Markdown 文件，命名格式：

```
YYYY-MM-DD_HH-MM_<工具名>_<用途简述>.md
```

例如：
- `2026-01-24_14-30_ChatGPT_文献检索.md`
- `2026-01-25_09-15_Copilot_代码调试.md`

## 日志内容模板

```markdown
# AI 交互日志

**日期时间**：YYYY-MM-DD HH:MM
**使用工具**：[工具名称及版本]
**使用者**：[团队成员姓名/编号]
**用途**：[简要说明]

## 输入 (Prompt)

[粘贴发送给 AI 的完整提示]

## 输出 (Response)

[粘贴 AI 的完整响应，或关键摘要]

## 人工处理

[说明如何使用/验证/修改 AI 输出]

## 最终采纳

- [ ] 直接采纳
- [ ] 修改后采纳
- [ ] 仅作参考
- [ ] 未采纳

**说明**：[简要解释采纳/不采纳原因]
```

## 注意事项

1. **及时记录**：交互后尽快记录，避免遗忘细节
2. **诚实完整**：如实记录，包括未采纳的交互
3. **隐私保护**：不要记录个人敏感信息
4. **定期汇总**：比赛结束前整理至 `12_ai_tool_report.tex`
